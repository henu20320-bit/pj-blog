# GitHub 主页 DIY 指南

GitHub 个人主页其实是一个很适合低成本展示自己的地方。很多人会把它当成一个仓库首页，但如果认真做一下，它完全可以承担“个人名片 + 技术展示页”的作用。

## GitHub 主页的基本原理

GitHub 个人主页的核心做法其实很简单：

1. 创建一个和自己 GitHub 用户名完全相同的仓库
2. 在这个仓库里写 `README.md`
3. GitHub 会把这份 README 直接展示到你的个人主页顶部

也就是说，主页本质上就是一份特殊仓库的 README。

## 快速开始

### 第一步：创建同名仓库

假设你的 GitHub 用户名是 `example`，那仓库名也要叫：

```text
example/example
```

用户名和仓库名必须完全一致，这一步不能错。

### 第二步：创建 README.md

在 `main` 分支下新建 `README.md`，然后 GitHub 就会把它识别成你的个人主页内容。

### 第三步：写最基础的个人信息

刚开始不用追求复杂，先把最核心的信息写清楚：

- 你是谁
- 你在做什么
- 你擅长什么
- 你最近在学什么
- 怎么联系你

## README 里最值得放的内容

### 1. 自我介绍

这是最基础的一块，建议写得短一点，但要有辨识度。

可以包含：

- 当前方向
- 关注技术栈
- 正在做的项目
- 联系方式

### 2. 技术栈

把自己会的内容分组列出来会比堆一串词更清晰。

比如：

- 编程语言
- 框架与库
- 工具链
- 云平台或部署能力

### 3. 项目展示

不要一股脑把所有仓库都堆进去，挑 2 到 4 个最能代表你的项目就够了。

每个项目最好附一句说明：

- 它解决了什么问题
- 你负责了什么
- 它的亮点是什么

### 4. GitHub 统计卡片

很多人会加统计卡片，这个没问题，但建议把它当“辅助信息”，不要让整个主页全是卡片。

统计卡片可以增强氛围，但真正能让人记住你的，还是文字表达和项目内容。

## 如果想更花一点，可以做什么

### 贪吃蛇贡献图动画

这是比较常见的一种玩法，通过 GitHub Actions 定时生成贡献图动画，再把 SVG 放到 README 里。

它的实现思路通常是：

1. 用 Action 定时跑任务
2. 根据贡献图生成蛇形动画 SVG
3. 推送到指定分支
4. 在 README 中引用生成结果

这种方式很适合增加主页的动态感。

### 3D 贡献图

另一种更偏展示效果的方案，是生成 3D 风格的贡献图。

这种内容视觉上会更吸睛，但也更建议放在 README 靠后的位置，不要压过真正重要的信息。

## 做主页时最常见的问题

### 1. 内容太花，但没有重点

很多主页第一眼很炫，但看完之后并不知道这个人到底擅长什么。

所以视觉效果可以有，但结构一定要清楚。

### 2. 项目太多，反而没有记忆点

项目展示不是越多越好。能被快速理解、快速记住，比堆数量更重要。

### 3. 只放卡片，不写人话

统计图、徽章、图标这些都只是配料。真正决定主页质量的，还是你有没有把自己讲明白。

## 我更推荐的一种主页思路

如果是为了求职、展示或者建立个人技术形象，我会更推荐这个结构：

1. 开头一句简洁定位
2. 一段简短自我介绍
3. 当前在做什么
4. 核心技术栈
5. 代表项目
6. 联系方式
7. 最后再补统计图或动态装饰

这样做的好处是信息层次清楚，别人打开主页几秒内就能知道你是谁。

## 小结

GitHub 主页 DIY 的关键，不在于堆多少视觉效果，而在于用最短路径把你自己展示清楚。README 只是载体，真正有价值的是你如何组织信息、如何讲述自己的方向和项目。
