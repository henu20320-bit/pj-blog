# AI Agent 入门梳理

这几年大家一提到 AI，最先想到的往往是大模型。但如果把视角再往前推一步，会发现真正决定“它能不能做事”的，往往不是模型本身，而是它被组织成了什么样的系统。

这也是为什么越来越多人开始把注意力从“模型”转向“AI Agent”。

## 什么是大语言模型

大语言模型可以理解成一个经过大规模训练的语言系统。它擅长做的事情包括：

- 理解自然语言输入
- 生成连贯文本
- 总结、改写、翻译
- 在一定上下文里完成推理

但模型本身更像“大脑”或者“推理核心”，它并不天然等于一个完整的工作系统。

## 什么是 AI Agent

AI Agent 可以理解成“带目标、能行动、会根据环境调整行为的 AI 系统”。

和单纯问答式模型相比，Agent 更强调这几件事：

- 感知输入
- 设定或接收目标
- 规划步骤
- 调用工具
- 执行动作
- 根据结果继续修正

所以如果说大模型擅长“回答”，那 Agent 更接近“做事”。

## 为什么 Agent 这么重要

因为现实任务通常不是一次问答就结束的。

比如一个完整任务可能需要：

1. 先理解需求
2. 再拆分步骤
3. 去检索资料
4. 调工具执行
5. 根据结果继续调整
6. 最后整理输出

单靠一个模型回复，很难把这个过程稳定串起来；但 Agent 恰好就是为了补这部分能力而生的。

## 搭建个人知识库为什么常和 Agent 放在一起讲

因为很多 Agent 的效果都依赖“外部知识”。

如果模型只依赖训练时学到的内容，它在处理你自己的文档、笔记、项目资料时很容易不够准。这个时候，就需要把你自己的知识接进来。

常见做法包括：

- 建个人知识库
- 把笔记、文档、PDF、网页资料统一整理
- 通过检索把相关内容喂回模型

这就是很多人开始同时关注知识库工具和 Agent 平台的原因。

## 什么是 RAG

RAG 是 `Retrieval-Augmented Generation`，通常翻译成“检索增强生成”。

它的核心思路很简单：

- 先检索与问题相关的资料
- 再让模型基于这些资料生成回答

这么做的好处很明显：

- 回答更贴近你自己的资料
- 降低胡编的概率
- 更适合知识库问答和资料型场景

如果把大模型看作发动机，那 RAG 更像给它接上了一套更可靠的外部信息供应系统。

## Agent 的能力通常会经历哪些阶段

我自己更愿意把它粗略分成三个阶段。

### 第一阶段：提示词驱动

这个阶段的系统主要靠预设提示词工作。

优点是简单、上手快；缺点是灵活性有限，任务一复杂就容易露出边界。

### 第二阶段：工作流驱动

这个阶段会把任务拆成多个步骤，每一步可能配不同模型、不同工具或不同规则。

这种方式已经比单提示词强很多，尤其适合稳定流程型任务。

### 第三阶段：更完整的智能体

更成熟的 Agent 不只是执行固定流程，而是会：

- 主动拆任务
- 判断该用什么工具
- 根据结果调整路径
- 在目标完成后收尾

这也是大家通常说的“更像真正智能体”的方向。

## 常见平台和工具怎么看

如果只是想先入门，不需要一开始就搭特别复杂的系统。可以先从这几类工具理解生态：

- 模型聚合类客户端
- 个人知识库工具
- 工作流编排平台
- Agent 开发平台

关键不在于哪个平台最强，而在于你到底要解决什么问题：

- 是做问答
- 是做资料检索
- 是做自动化流程
- 还是做多步骤任务执行

## 我自己的理解

我越来越倾向于把 Agent 看成一个组合系统，而不是某一个单点能力。

它至少包含四层：

- 模型
- 记忆或知识
- 工具调用
- 任务流程

模型决定“想”，工具决定“做”，知识决定“准不准”，流程决定“能不能持续完成任务”。

## 小结

大模型解决的是“能不能理解与生成”，而 AI Agent 解决的是“能不能围绕目标持续完成任务”。如果只停留在模型层，很多场景只能做到好看；一旦接入知识、工具和流程，才真正开始具备实用价值。
